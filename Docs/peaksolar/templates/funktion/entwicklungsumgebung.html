Insomnia

Insomnia ist – ähnlich wie Postman – ein Tool zum Testen von HTTPS-Requests (Integrationstests). Da die Datenmodellierung für GraphQL besonders wichtig ist, kann man sogenannte Introspection-Anfragen stellen.

Diese liefern Informationen über den erforderlichen Aufbau des an den Server zu sendenden JSON-Formats.

Hier ist ein Beispiel für eine Introspection-Anfrage für das "Customer"-Modell von HERO:


query {
   __type(name: "Customer") {
   name
     fields {
       name
       type {
         name
         kind
       }
     }
   }
 }


Einige der Datenstrukturen werden auch auf der eigenen Hero Dokumentation beschrieben, diese ist aber nicht allzu umfassend. https://support.hero-software.de/hc/de/articles/7439986901020-GraphQL-Use-Cases


XAMPP

API-Anfragen sind grundsätzlich von der verwendeten Programmiersprache unabhängig. Da WordPress jedoch auf PHP basiert, bietet es sich an, PHP für die Anfragen zu nutzen. Ein einfacher PHP-Server sowie eine lokale MySQL-Datenbank werden durch XAMPP bereitgestellt.

Damit das Hot-Module-Reloading (HMR) mit PHP funktioniert, muss eine Browser-Erweiterung verwendet werden, die als Proxy zwischen XAMPP und dem "Live-Server"-Plugin agiert:

  • Live-Server Web Extension: Chrome Web Store
  • Live-Server Plugin für VS Code: VS Code Marketplace

Weitere Informationen zur Einrichtung findest du hier:

Live-Server Web Extension – Setup.

Während der Entwicklung trat ein Problem mit der lokalen MySQL-Datenbank des Projekts auf. Falls sich MySQL nicht starten lässt, kann es helfen, die folgenden Dateien im \Xampp\mysql\data Ordner umzubenennen:

  • ib_logfile0
  • ib_logfile1
  • aria_log_control
